Spoofing - Technological

The other 7 points of attack on a finger print scanner are much more complex and technological.

2. Bypassing to sensor to resubmit previously stored digitized biometrics signals.
In order to attack the system at this point, the attacker would need to submit an old copy of a finger print image and pass it off as data from the scanner.

3. Overriding the feature extraction process
To override the feature extraction process, a Trojan horse would be used that makes the feature extraction process produce features predetermined by the attacker. This way any finger print could be passed off as the correct finger print as long as the feature extraction process produces the features that the matcher will be looking for.

4. Tampering with the biometric feature representation after extraction
After the features are extracted, instead of overriding before this process, the attacker can override the results and change what the results are. Attacks at this point in the verification process are very difficult because the stages of feature extraction and matches are often inseparable, so changing data between the steps is quite difficult. The best way to go about doing this type of attack would be to “snoop” on the TCP/IP stack and alter the packets in the case that information is transmitted to a remote matcher over the internet.

5. Corrupting the matcher
The matcher can be corrupted, especially if it is a remote matcher, so that it only returns the preselected match scores for the finger print—labelling every finger print a match. This can also be done so only certain fingerprints yield a match even though they are not.

6. Tampering with stored templates used by the matcher
The matcher uses templates that are either remote or local and possible on multiple servers in order to produce a match. These can be manipulated in order to make the matcher determine an incorrect fingerprint a match or deny a correct one.

7. Attacking the channel between the stored templates and the matcher
The templates themselves do not need to be manipulated. Since the templates are sent through a communication channel, the templates can be intercepted and modified en route to the matcher.

8. Overriding the final decision
The last possible point of attack is the final decision. The attacker can override this final decision that the matcher came to whether or not everything else so far in the process was wrong. If the final decision is not secure, the entire process of the fingerprint scanner, even if extremely accurate, becomes pointless.
